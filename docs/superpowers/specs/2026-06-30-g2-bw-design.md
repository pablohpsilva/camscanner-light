# G2 B&W Filter — Implementation Design

**Date:** 2026-06-30
**Status:** Approved (design)
**Sub-project:** 1 — Core scan pipeline
**Depends on:** G1 (`ImageEnhancer`/`NoneEnhancer`/`GrayscaleEnhancer`, `image` 4.9.1)
**Feeds:** G3 (Color/Auto), G4 (filter picker UI)
**Step in roadmap:** G2 — B&W binarization filter (second of G. Enhancement series)

## Purpose

Add a black-and-white (binarization) filter to the scan pipeline. After flattening
and optional perspective correction, users can binarize the page to crisp black
pixels on white — ideal for text-heavy documents and produces the smallest JPEG
output. Uses Otsu's automatic threshold (no magic numbers) run in a `compute()`
isolate. A new icon button in the review screen's AppBar gives immediate user control,
mutually exclusive with the grayscale toggle from G1.

## Scope

**In scope:**
- `BwEnhancer` strategy class implementing `ImageEnhancer`
- Otsu's histogram-based threshold (`_otsuThreshold`) as a private top-level function
- Second icon button (`Key('bw-toggle')`) in `CaptureReviewScreen`'s AppBar
- Replace `bool _grayscale` with `_EnhancerMode` enum (none / grayscale / bw) in `CaptureReviewScreen`
- Mutual exclusion between grayscale and B&W buttons (implicit in enum)

**Out of scope:**
- DB column for enhancement mode (G4)
- Non-destructive re-selection after save (G4)
- Live B&W preview (G4)
- Color, Auto/Magic filters (G3)
- Filter picker UI (G4)

## Architecture

### New: `lib/features/library/bw_enhancer.dart`

OCP — nothing modified, only added. Parallel to `grayscale_enhancer.dart`:

```dart
import 'dart:typed_data';
import 'package:flutter/foundation.dart' show compute;
import 'package:image/image.dart' as img;
import 'image_enhancer.dart';

class BwEnhancer implements ImageEnhancer {
  const BwEnhancer();

  @override
  Future<Uint8List> enhance(Uint8List bytes) => compute(_bwFn, bytes);
}

Uint8List _bwFn(Uint8List bytes) {
  try {
    final decoded = img.decodeImage(bytes);
    if (decoded == null) return bytes;
    // bakeOrientation: EXIF scrubber keeps the Orientation tag; encodeJpg
    // strips EXIF, so orientation must be baked into pixels first.
    // For already-baked flat bytes (post-warp), this is a safe no-op.
    final oriented = img.bakeOrientation(decoded);
    img.grayscale(oriented);                   // luminance-weighted, mutates in place
    final t = _otsuThreshold(oriented);        // automatic split — no magic numbers
    final bw = img.threshold(oriented, t);     // binarize; returns new Image
    return Uint8List.fromList(img.encodeJpg(bw, quality: 92));
  } catch (_) {
    return bytes;
  }
}

// Otsu's method: maximises inter-class variance between background and foreground.
// Works on an already-grayscale Image — reads pixel.r (all channels equal).
int _otsuThreshold(img.Image src) {
  final hist = List<int>.filled(256, 0);
  for (final px in src) {
    hist[px.r.toInt()]++;
  }
  final total = src.width * src.height;
  double sum = 0;
  for (var i = 0; i < 256; i++) sum += i * hist[i];

  double sumB = 0, maxVar = 0;
  int wB = 0, threshold = 128;
  for (var t = 0; t < 256; t++) {
    wB += hist[t];
    if (wB == 0) continue;
    final wF = total - wB;
    if (wF == 0) break;
    sumB += t * hist[t];
    final mB = sumB / wB;
    final mF = (sum - sumB) / wF;
    final v = wB.toDouble() * wF * (mB - mF) * (mB - mF);
    if (v > maxVar) {
      maxVar = v;
      threshold = t;
    }
  }
  return threshold;
}
```

`img.grayscale()` uses luminance-weighted conversion and mutates in place.
`img.threshold()` returns a new `Image` (checked against image 4.9.1 API).
Quality 92 matches `PerspectiveWarper` and `GrayscaleEnhancer`.

### Modified: `lib/features/scan/capture_review_screen.dart`

**New file-private enum** (replaces `bool _grayscale`):

```dart
enum _EnhancerMode { none, grayscale, bw }
```

**State field:**
```dart
_EnhancerMode _mode = _EnhancerMode.none;
```

**AppBar actions** — two mutually exclusive icon buttons:

```dart
actions: [
  // Grayscale — updated to use _mode instead of _grayscale bool
  IconButton(
    key: const Key('grayscale-toggle'),
    icon: Icon(_mode == _EnhancerMode.grayscale
        ? Icons.filter_b_and_w
        : Icons.filter_b_and_w_outlined),
    tooltip: _mode == _EnhancerMode.grayscale ? 'Grayscale on' : 'Grayscale off',
    onPressed: () => setState(() => _mode =
        _mode == _EnhancerMode.grayscale
            ? _EnhancerMode.none
            : _EnhancerMode.grayscale),
  ),
  // B&W — new. Uses color to indicate active state (no guaranteed _outlined
  // variant for Icons.contrast across all Flutter versions).
  IconButton(
    key: const Key('bw-toggle'),
    icon: Icon(
      Icons.contrast,
      color: _mode == _EnhancerMode.bw
          ? Theme.of(context).colorScheme.primary
          : null,
    ),
    tooltip: _mode == _EnhancerMode.bw ? 'B&W on' : 'B&W off',
    onPressed: () => setState(() => _mode =
        _mode == _EnhancerMode.bw
            ? _EnhancerMode.none
            : _EnhancerMode.bw),
  ),
],
```

Mutual exclusion is implicit in the enum — setting `_EnhancerMode.bw` naturally
deactivates grayscale, and vice versa.

**Accept button** — Dart 3 exhaustive switch:

```dart
onPressed: widget.saving
    ? null
    : () => widget.onAccept(
          _corners,
          switch (_mode) {
            _EnhancerMode.grayscale => const GrayscaleEnhancer(),
            _EnhancerMode.bw       => const BwEnhancer(),
            _EnhancerMode.none     => const NoneEnhancer(),
          },
        ),
```

**`onAccept` signature is unchanged** — `void Function(CropCorners, ImageEnhancer)`.
`CameraScreen`, `SaveController`, and `DriftDocumentRepository` need no changes.

The internal `bool _grayscale → _EnhancerMode _mode` replacement is transparent
to existing G1 widget tests: they interact via `Key('grayscale-toggle')` and check
the `onAccept` callback — behaviour is identical.

## Data flow

```
User taps B&W toggle (review screen)
  → _mode = _EnhancerMode.bw

User taps Accept
  → widget.onAccept(_corners, BwEnhancer())   [switch on _mode]
  → CameraScreen._onAccept(image, corners, BwEnhancer())
  → SaveController.save(image, corners: corners, enhancer: BwEnhancer())
  → DriftDocumentRepository.createFromCapture(...)
      → scrub → warp (if cropped) → BwEnhancer.enhance(bytes) [compute isolate]
      → Otsu threshold → binary JPEG → write to disk
```

Full-frame and cropped paths are handled identically to G1 — the pipeline is
already wired.

## Deliverable (user-testable)

A B&W toggle button in the review screen's AppBar, alongside the grayscale button.
Tap it to enable B&W binarization, then Accept — the saved page is crisp black
text on white. Tapping the other button deactivates the current one.

**You can test it by:**
1. Scan a printed text document. On the review screen, tap the contrast icon
   (B&W). Tap Accept — the saved page should be crisp black-and-white.
2. Tap the grayscale icon instead — only grayscale should activate (B&W deactivates).
3. Tap an active button a second time — it should deactivate (no filter).
4. Confirm Retake, Reset, and crop overlay work normally (no regression).

      runs off the UI thread, bakes EXIF orientation — *unit*
