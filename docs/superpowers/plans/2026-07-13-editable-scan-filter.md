# Editable Scan Filter (Non-Destructive) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let the user change a scanned page's enhancement filter (Auto / Original / Color / Grayscale) after capture, losslessly, by storing the pristine *unfiltered* original and treating the enhancer as composable page metadata.

**Architecture:** The enhancer becomes the third composable page-metadata field alongside `rotationQuarterTurns` and `corners`. The base image file becomes truly pristine (unfiltered); the displayed "flat" is regenerated by `DriftDocumentRepository._writeFlat` as **enhance ∘ rotate ∘ crop** from the base, routed through the existing `PageProcessor.process(bytes, corners, mode)` pipeline. A new dedicated Filter screen (mirroring the scan-review screen) lets the user pick a mode; the page-viewer applies it via the existing single-flight + image-epoch refresh.

**Tech Stack:** Flutter, Drift (sqlite) + FTS5, `package:image`, `compute` isolates, `bdd_widget_test`.

## Global Constraints

- All Flutter commands run from `apps/mobile/`.
- Bump `schemaVersion` and add an `onUpgrade` step for the new column.
- Store RELATIVE image paths only; **never** write the base file during an edit (the base is pristine, write-once).
- Keep all full-res decode/rotate/warp/enhance work OFF the UI isolate.
- Edits stay single-flight, with the `_imageEpoch` ValueKey display refresh so a same-path regenerated flat re-decodes.
- `EnhancerMode` enum order is FIXED: `enum EnhancerMode { none, grayscale, auto, color }` → stored as its index `none=0, grayscale=1, auto=2, color=3`.
- Legacy pages (pre-v8) have no unfiltered original and default to `enhancerMode = none`; changing their filter re-filters the already-filtered base (may stack) — an accepted, documented gap.
- TDD + BDD; nothing is "done" until green on a real Android device AND a real iOS device (or a named, explicit gap). Host tests via `flutter test`; regenerate generated code with `dart run build_runner build --delete-conflicting-outputs`.

```dart
  /// Regenerates the display ("flat") derivative from the PRISTINE base by
  /// applying enhance ∘ rotate ∘ crop, and returns the new flat's relative path
  /// (or null when the display equals the base: no rotation, no crop, no
  /// filter). NEVER writes the base. [corners] are in the display frame
  /// (post-rotation).
  Future<String?> _writeFlat({
    required String relativeImagePath,
    required int quarterTurns,
    required CropCorners corners,
    required EnhancerMode mode,
    required String? existingFlatRel,
  }) async {
    final baseBytes = await _fileStore
        .absoluteFor(relativeImagePath)
        .readAsBytes();
    final isFullFrame = corners == CropCorners.fullFrame;

    // Fast path: the display equals the pristine base — no image work at all.
    if (quarterTurns == 0 && isFullFrame && mode == EnhancerMode.none) {
      await _deleteFlatIfPresent(existingFlatRel);
      return null;
    }

    // Rotation needs a full-res decode+rotate+re-encode — run it OFF the UI
    // isolate (`compute`). quarterTurns == 0 skips it (the processor bakes EXIF
    // orientation itself), so it consumes the base bytes directly.
    Uint8List? rotatedBytes;
    if (quarterTurns != 0) {
      rotatedBytes = await compute(
        rotateAndBakeJpeg,
        RotateJpegArgs(baseBytes, quarterTurns),
      );
      if (rotatedBytes == null) {
        throw const DocumentSaveException('regenerate: undecodable base image');
      }
    }
    final input = rotatedBytes ?? baseBytes;

    Uint8List? flatBytes;
    if (isFullFrame) {
      if (mode == EnhancerMode.none) {
        // quarterTurns != 0 here (pure pass-through handled above).
        flatBytes = rotatedBytes;
      } else {
        // Full-frame enhancement; fall back to the un-enhanced frame on failure
        // so a page is never lost.
        flatBytes =
            await _processor.process(input, CropCorners.fullFrame, mode) ??
            input;
      }
    } else {
      // Cropped: the processor warps + enhances in one pass (or two-step for a
      // stubbed warper). Fall back to the rotated-only image if it makes nothing.
      flatBytes = await _processor.process(input, corners, mode) ?? rotatedBytes;
    }

    if (flatBytes == null) {
      await _deleteFlatIfPresent(existingFlatRel);
      return null;
    }
    final flatRel =
        existingFlatRel ?? _fileStore.flatForImage(relativeImagePath);
    await _fileStore.writeRelative(flatRel, flatBytes);
    return flatRel;
  }

  Future<void> _deleteFlatIfPresent(String? rel) async {
    if (rel == null) return;
    try {
      await _fileStore.absoluteFor(rel).delete();
    } on FileSystemException {
      /* already gone — fine */
    }
  }
```

  @override
  Future<void> updatePageEnhancer(
    int documentId,
    int position,
    EnhancerMode mode,
  ) async {
    final row =
        await (_db.select(_db.pages)..where(
              (t) =>
                  t.documentId.equals(documentId) & t.position.equals(position),
            ))
            .getSingleOrNull();
    if (row == null) {
      throw DocumentSaveException(
        'updatePageEnhancer: no page ($documentId, $position)',
      );
    }
    final corners =
        CropCorners.tryParse(row.corners) ?? CropCorners.fullFrame;
    final flatRel = await _writeFlat(
      relativeImagePath: row.relativeImagePath,
      quarterTurns: row.rotationQuarterTurns,
      corners: corners,
      mode: mode,
      existingFlatRel: row.flatRelativePath,
    );
    await (_db.update(_db.pages)..where((t) => t.id.equals(row.id))).write(
      PagesCompanion(
        enhancerMode: Value(mode.index),
        flatRelativePath: Value(flatRel),
      ),
    );
    await (_db.update(_db.documents)..where((d) => d.id.equals(documentId)))
        .write(DocumentsCompanion(modifiedAt: Value(_clock().toUtc())));
  }

## Notes for the implementer

- **Do NOT `git add -A` / `git add .`** — the working tree carries a long-lived unrelated WIP pile. Scope every `git add` to the exact paths listed in each task (per [[pre-existing-wip-pile-git-add-hazard]]).
- The composable-edit invariant is load-bearing: an edit NEVER writes `relativeImagePath` (the base). Only `createFromCapture` / `addPageToDocument` / `replacePage` (fresh captures) write it. If a change makes an edit write the base or derive the flat from the flat, the "can only edit once" bug returns.
- `img.decodeImage` auto-applies EXIF orientation in image 4.x; the processor/rotate paths also `bakeOrientation` as belt-and-suspenders. Don't "fix" this by removing the bake.
- OpenCV-dependent host failures are environmental (`libdartcv` doesn't load under plain `flutter test`); the tests in this plan use `HybridWarper`/`FakeImageWarper` and `package:image` only, so they run on the host without OpenCV.
