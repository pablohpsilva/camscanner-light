# C2 — In-app PDF preview Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Tapping the viewer's Export action regenerates the PDF (C1) and opens an on-device `PdfPreviewScreen` that renders it with pinch-zoom — superseding C1's "PDF saved" SnackBar.

**Architecture:** A new `PdfPreviewScreen` renders `export.pdf` via `pdfx` (`PdfViewPinch`). Because `pdfx`'s `errorBuilder` does NOT catch `openFile` failures (spike-proven), the screen opens the doc via an **injectable opener** in `initState` try/catch → explicit loading/error/loaded states. The viewer's `_exportPdf` navigates to it on success. C1's export-success behavior is **replaced** (BDD + gate), not evolved.

**Tech Stack:** Flutter 3.44.4, `pdfx: ^2.9.0` (renderer, no network dep), `pdf` (C1 generator), `bdd_widget_test`, Nx.

## Global Constraints

- **Privacy spine:** the PDF is rendered **on-device** by `pdfx` (no network package in its dep tree); nothing shared/uploaded/sent off-device.
- **`pdfx` error handling is in-screen, NOT via `errorBuilder`** (spike-proven: `PdfViewPinch.errorBuilder` does not catch `PdfDocument.openFile` failures). Open via the injectable opener in `initState` try/catch.
- **`pdf`/`pdfx` `PdfDocument` collision:** both packages export `PdfDocument`. Any file importing BOTH must `import 'package:pdf/...' hide PdfDocument;`. (C2's files are structured to avoid importing both where possible.)
- **Viewer→preview host test uses `tester.pump()`, not `pumpAndSettle()`** — a real-opener `PdfPreviewScreen` hits the pdfx MethodChannel in host; settle could hang.
- **No schema change** (`schemaVersion` stays 1).
- **REAL_DEVICE deferred-with-sign-off:** pixel-correct render + zoom gesture (criterion 7) is opt-in, not gated.
- **Personal `apps/mobile/ios/Runner.xcodeproj/project.pbxproj` stays uncommitted** — `pdfx` reintroduces CocoaPods on iOS; the `Podfile` is **auto-generated by `flutter` at iOS build** (the verify gate's `pod install` handles it) and is **not committed** — pod churn stays in the uncommitted `pbxproj`, as today. Never `git add` the pbxproj.

Create `apps/mobile/lib/features/library/pdf_preview_screen.dart`:

```dart
import 'package:flutter/material.dart';
import 'package:pdfx/pdfx.dart';

/// Opens a PDF file and returns its [PdfDocument]. Injectable so host tests can
/// drive loading/error without the native plugin. Default = pdfx's real opener.
typedef PdfOpener = Future<PdfDocument> Function(String path);

/// Renders [pdfPath] on-device with pinch-zoom ([PdfViewPinch]). The doc is
/// opened in initState via [opener] wrapped in try/catch — pdfx's errorBuilder
/// does NOT catch openFile failures (spike-proven), so error handling lives
/// here, surfacing explicit loading / error / loaded states.
class PdfPreviewScreen extends StatefulWidget {
  final String pdfPath;
  final String name;
  final PdfOpener opener;
  const PdfPreviewScreen({
    super.key,
    required this.pdfPath,
    required this.name,
    this.opener = PdfDocument.openFile,
  });

  @override
  State<PdfPreviewScreen> createState() => _PdfPreviewScreenState();
}

class _PdfPreviewScreenState extends State<PdfPreviewScreen> {
  PdfControllerPinch? _controller;
  bool _loading = true;
  bool _error = false;

  @override
  void initState() {
    super.initState();
    _open();
  }

  Future<void> _open() async {
    try {
      final doc = await widget.opener(widget.pdfPath);
      if (!mounted) {
        doc.close();
        return;
      }
      setState(() {
        _controller = PdfControllerPinch(document: Future.value(doc));
        _loading = false;
      });
    } catch (_) {
      if (!mounted) return;
      setState(() {
        _loading = false;
        _error = true;
      });
    }
  }

  @override
  void dispose() {
    _controller?.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text(widget.name)),
      body: _loading
          ? const Center(
              key: Key('pdf-preview-loading'),
              child: CircularProgressIndicator())
          : _error
              ? const Center(
                  key: Key('pdf-preview-error'),
                  child: Text("Couldn't open the PDF."))
              : PdfViewPinch(
                  key: const Key('pdf-preview-view'),
                  controller: _controller!,
                ),
    );
  }
}
```

    try {
      final file = await widget.repository.exportPdf(widget.documentId);
      if (!mounted) return;
      Navigator.of(context).push(
        MaterialPageRoute<void>(
          builder: (_) =>
              PdfPreviewScreen(pdfPath: file.path, name: widget.name),
        ),
      );

      );
    } finally {
      if (mounted) setState(() => _exporting = false);
    }
  }
